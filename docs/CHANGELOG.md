- Three-reviewer pass (bughunter + backend + security) on the multi-route changes, all findings fixed. MUST FIX (backend): a `javascript:`/`mailto:`/`tel:` anchor has an opaque origin, so the naive off-origin test dropped it as an "external link never fired" — losing the common `javascript:void(0)` button idiom AND inflating coverage with a control never exercised; the skip is now gated on http(s) cross-origin only (`isOffOriginHttp` in `scope.mjs`). HIGH (security): a same-origin `//evil.com` path key reconstructed via `new URL(key, origin)` as a PROTOCOL-RELATIVE url (→ https://evil.com/) and `gotoGated` allows public hosts — escaping origin scope onto attacker infra; `routeKey` now collapses leading slashes to one and `persistentStep` refuses any reconstructed navUrl that is not same-origin (defense-in-depth). SHOULD FIX (backend): `fillTarget`'s `fill()` calls were unbounded (a hidden/disabled sibling input hung the 30s default timeout — the same stall the click gate kills) → both bounded to 5s; an off-origin JS-redirect landing now reports the in-scope start route, not the foreign pathname. LOW (security): `report`'s `oneLine` now strips C0/C1 control bytes before rendering agent-/error-derived text (no terminal-escape injection), and `purpose` is sanitized; the internal validation hostname was genericized out of this changelog. New revert-proven guards: `scope` unit (leading-slash collapse + `isOffOriginHttp` scheme gate) and `tests/live/scheme-gate.test.mjs` (a javascript: anchor is fired, not skipped). ACCEPTED (documented, mostly pre-existing): a same-origin control whose JS onclick redirects off-origin still navigates before we can refuse it — inherent to driving untrusted pages (any visited page can script-navigate on load); the after-merge guard keeps it out of the graph and the next act re-navigates back. 63 tests green.
- Real-site generalization: MULTI-ROUTE recon (run on any same-origin site, collect more than the entry page; design by cto/Fable, code Opus). `actStep` now derives the after-snapshot route from `page.url()` (the route the act LANDED on) and merges revealed controls under it — simultaneously the correctness fix for nav acts AND the multi-route discovery engine: new routes' controls enter the graph with the right `node.route` and the revert-proven global template frontier hands them out unchanged. The persistent step re-navigates per act to the TARGET's own `node.route` (a control first seen on /products is reached by loading /products). New `lib/recon/scope.mjs` owns route identity (`routeKey` — drop query + plain #anchor, keep path-like SPA hash, keep the CONCRETE navigable path) and origin scope (`sameOrigin`, RFC 6454 scheme+host+port). Off-origin links are NEVER fired: `actStep` reads an `<a>`'s href and records an off-origin one as `external` (effect `external-link`, counted, not clicked) — the SSRF gate only guards our navigations, so this is the scope boundary for acted links. `frontierStats.routes` reports routes mapped (surfaced in `report`). New guards: `scope` unit (routeKey normalization + sameOrigin port sensitivity) and one live crawl (`tests/live/multi-route.test.mjs`) proving all three mechanisms — multi-route reach, landed-route attribution, off-origin never fired (partner server gets zero hits) — each revert-proven. 60 tests green. Deferred to the stay-on-page ADR: BFS route queue + per-route budget, eager href harvest, per-(route×template) causal keying, and overlay/cookie-consent dismissal (a consent banner still leaves controls NOT_VISIBLE).
- Real-site hardening #1 (found by a live public-site validation run): the fire path stalled the full 30s Playwright actionability timeout on every DOM-present-but-invisible control (a responsive layout's mobile menu is `display:none` on desktop, yet dom-snapshot captures it — by design, it does not filter by visibility). On a real site that alone cripples a run. `actStep` now `isVisible()`-fast-fails such a control with `NOT_VISIBLE` before the click (and bounds the click itself to 5s so a transiently-unclickable control fails in seconds, not 30). `observe` gains a `not-visible` effect that `markUnreachable`s — honest-coverage parity, so a page full of hidden/duplicated controls does not inflate the explored denominator. `recon.md` maps `NOT_VISIBLE ⇒ not-visible` in its effect derivation. New revert-proven live guard (`tests/live/not-visible.test.mjs`: hidden `#hid` rejects `NOT_VISIBLE` in <2s while the visible sibling still acts) + unit guard (`observe.test.mjs`). 57 tests green.
