# bughunter

Bug-hunting agent for web apps, built on Claude Code. Ground-up rebuild of the sibling project bughunt-agents (decisions.md records why the old foundation was abandoned). Runs inside Claude Code — no API keys.

<invariants>
- IMPORTANT: READ docs/GOAL.md — it is the standing definition of done and it overrides any local notion of "good enough". THE SCRIPT IS THE BRAIN, THE AGENT IS THE HANDS: the script owns the count (total / studied / outstanding) and drives the loop until the outstanding list is empty; the agent only executes. ONE TOUCH IS NOT A STUDY — an element is understood only after the variations that could distinguish it, INCLUDING the ones expected to fail (empty, over the declared limit, wrong shape), and a form is studied INCREMENTALLY (submit empty → one field → two → then invalid values), because each submit asks a different question. ## Pipeline
Two phases over one graph. Phase 1 (recon, "perceptron loop"): each context window studies 2-5 NEW elements, acts with a causal token, writes observations to the graph, loops. Phase 2 (Fable/Opus): reads the compact graph, designs Given/When/Then cases, generates specs; coverage feeds gaps back. Full shape in docs/ARCHITECTURE.md.

Status: the keystone (`whats-new`), the Phase-1 loop CORE, the semantic recon layer, AND the shared browser daemon are built and proven green. Loop core: `frontier` + `recon-loop` (pure driver, step+judge injected) + `recon-run` (one browser for the whole crawl, re-navigate per act). Semantic layer (the LLM judge): the `recon` subagent (`.claude/agents/recon.md`, Sonnet) drives thin CLIs — `frontier-cli` (emit the receptive field), `whats-new` (act+capture), `observe` (write purpose/danger/effect + mark explored, gated by a `danger-floor` backstop that refuses firing destructive/logout/payment controls). Shared browser: `recon-session --start` boots ONE chromium daemon per run; `whats-new`/`recon-run` `attach()` to it (one process, not one per act) and fall back to a cold launch when absent. Explored ⟺ observed in the agent path. Recon is MULTI-ROUTE: `actStep` attributes revealed controls to the route the act LANDED on (`routeKey(page.url())`), the persistent step re-navigates per act to the target's own `node.route`, and off-origin links are recorded but NEVER fired (`scope.mjs`) — so a run on any same-origin site collects more than the entry page. Phase 1 is RUNNABLE END-TO-END: `/recon <url>` (`.claude/commands/recon.md`) opens a debug-capture run, boots the admin viewer, brackets the daemon, loops the `recon` agent until the frontier drains, and prints `report` (honest coverage + routes mapped + causal control→endpoint map, the Phase-2 input). Cookie/consent overlays are dismissed before each snapshot/act (`overlays.mjs`), so a consent wall no longer leaves every underlying control `NOT_VISIBLE`. AUTHENTICATED RECON is built: a login PRE-STEP (`login.mjs`) submits operator-env creds ONCE, verifies the session, and persists a Playwright storageState (`state/storage-state.json`, 0600); `session.mjs contextOptions()` loads it at BOTH newContext sites (cold + daemon-attach) via `BUGHUNTER_STORAGE_STATE`, so every cold re-navigate crawls logged-in with the SAME graph/causal machinery. A set-but-missing state path fails loud, never a silent logged-out crawl. A route-level guard (`routeRefused`, `recon-run`) refuses to NAVIGATE to a destructive/auth/payment route (the GET `/logout` the name-only click gate cannot see) so an authed crawl never ends its own session. Creds come from `BUGHUNTER_LOGIN_USER`/`_PASS`, never argv or a committed file; login writes no debug trail (the credential frame is never captured). DEBUG CAPTURE (opt-in, set by `/recon`): with `BUGHUNTER_RUN_ID` set, every act writes a `{seq,ts,kind,payload}` trail (`lib/debug/trace.mjs`) — `route`/`frontier.emit`/`act`/`observe` events + per-step graph snapshots + before/after viewport key-frames (taken while the cause is `__idle__`, so they never perturb causal attribution). A zero-dep local `admin-server` (127.0.0.1, Host-guard, no-CORS, path-allowlisted) serves `admin.html`, a single page that replays the run: graph growth (scrubber), the agent's walk, what it tested (before/after screenshots + rect box), logs, and speed. REQUEST CAPTURE goes beyond method+URL: each causally-attributed request records response status/mimeType/resourceType/duration (a per-`requestId` CDP ledger joined to the kept fires), and — opt-in via `BUGHUNTER_CAPTURE_BODIES` + an active run — REDACTED request/response bodies written ONLY to the gitignored run trail (`lib/browser/redact.mjs` masks secrets at capture time; bodies never reach stdout/graph). STAY-ON-PAGE reach is built for BOTH the node-loop AND the agent path: a control revealed only by an in-page action carries an additive `node.reveal` reveal-path annotation (ZERO identity churn — gated by the read-only `lib/graph/identity-diff.mjs` probe), and BOTH `recon-run` (node loop) AND `whats-new` (the agent path `/recon` drives) reach it by REPLAY-FROM-RESET (re-nav/reset → replay the reveal-path clicks under `__idle__` → the measured act; the SAME `lib/recon/reveal-replay.mjs` `applyReveal` prologue, not forked), so a modal/dropdown control is genuine coverage on `/recon` too, not `unreachable-coldstart`. Replayability: an all-GET opener is replayable by default; a POST-that-READS opener (list/search fired over POST — the constant-URL SPA class, where the whole app lives under one URL and nav swaps content client-side) is replayable only when the AGENT judges it a read and passes `whats-new --opener-replayable` (`step.mjs` stamp = `allGet || openerReplayable`); the node loop never sets it, so its deterministic GET-only ceiling holds; a mutating opener's children stay honestly unreachable. A `REVEAL_CYCLE` guard refuses a cyclic reveal path, and every context uses a FIXED 1440×900 viewport (`contextOptions()`) for a reproducible NOT_VISIBLE denominator. FRAMEWORK-ID IDENTITY (INC.1) is built: `dom-snapshot.mjs isFrameworkNoiseId` rejects framework-generated wrapper ids (`rc-*`/`ant-*`/`radix-*`/`headlessui-*`/`mui-*` prefixes + hashed-hex-with-digit runs) from BOTH the selector-path anchor (`isGeneratedId`) and the sole-`#id` locator (`stableIdForLocator`), so Ant Design tabs (`rc-tabs-0-tab-N`) that fragmented into N per-reload-shifting templates collapse to ONE structural template with N addressable instances + a stable role+name locator — fixing the antd `NOT_VISIBLE`/`NO_INSTANCE` residual AND the stale-selector break in the reveal-replay chain (framework CLASSES like `ant-tabs-tab` are deliberately KEPT as the structural anchor). It is an INTENTIONAL identity re-key (the one class `identity-diff` protects — measured, not zero-gated; a no-op on non-framework fixtures) guarded by `graph.schemaVersion`=2 (`loadGraph` resets a legacy graph rather than co-mingling two id schemes). Fable (research: Crawljax/Judge/RIA/WebRLED; CTO: validated against the code) directed this: the compositional depth-N reveal a whole-SPA crawl needs is ALREADY built (`reveal-replay.mjs revealPathFor` + `step.mjs` stamp + `state-model.test.mjs`), so identity STABILITY — not a new depth mechanism — was the gap; the State node (INC.3 `graph.states{}`) is DEFERRED as reporting/dedup (not coverage), and amortized in-app DFS is rejected-for-now (it would break the `__idle__` causal discipline). DEPTH-2 PANEL REACH is built (Fable-CTO Option A): a control PRESENT at baseline but HIDDEN behind an overflow panel (an antd "…more") is discovered PATHLESS, so first-reveal-wins would lock it `NOT_VISIBLE`-unreachable — now `dom-snapshot` records a per-element `visible` (Playwright parity, additive/never identity), `mergeSnapshot` stamps write-once `hiddenWhenSeen` and FILLS + REOPENS a pre-existing hidden instance's reveal path when a STAMPED opener act makes it visible (clearing `explored`/`unreachable` so the frontier re-emits it), `step.mjs` flags a pure-uncover "…more" opener (revealing NO new instances) via the fill, and `persistentStep` re-reads `inst.reveal` from the graph so a same-batch opener-first ordering reaches the child; a tab with no opener stays honestly unreachable (fixture-proven, `panel-reach.test.mjs`; the live target reach number is separate validation). WHOLE-SITE ROUTE-FRONTIER (INC.1a node-loop core) is built: a BFS URL route queue lives INSIDE `graph.routes` as additive flags (`pending`/`pattern`/`siblings`/`unreachable`) — `lib/recon/route-frontier.mjs` harvests same-origin `a[href]` and SNAPSHOT-visits each page ONCE, so a control on a page beyond the element `OPENER_INSTANCE_CAP` (a wide nav's 9th tab, a listing's detail page) is reached. `harvestRoutes` is the ONE other `graph.routes` writer — METADATA only, NEVER elements/edges — and discovery is phantom-edge-safe BY CONSTRUCTION: it never `addTrigger`/`beginCause` and never opens a causal window, so navigation asserts only exists/reachable and adds ZERO edges (attribution stays token+initiator). SEED-THEN-LOOP: `recon-run` BFS-drains the discoverable queue up front (edge-free) before any act, then `reconLoop`'s injected `refill` hook drains routes discovered only by acting; a url-PATTERN census bound (`toUrlPattern`, the ONE masker) keeps ONE representative concrete route per pattern (a 50-row listing → 1 visit + 49 siblings folded); `routeRefused`/`sameOrigin`/`isOffOriginHttp`/SSRF (`navigateGated`, a Response-returning sibling of `gotoGated`) reused with NO bypass; a 404/redirect route is `markRouteUnreachable` (counted, never dropped). The route-frontier is now WIRED INTO THE AGENT PATH TOO (INC.1b): `whats-new` `harvestRoutes` at the SAME two `__idle__` points `persistentStep` uses (after the baseline snapshot; after the act's `endCause` — metadata-only, edge-free), and a new deterministic driver `lib/recon/route-cli.mjs --visit-next` (the agent-path twin of the node-loop's `refill`) navigates ONE pending route per call via `visitRoute` — the LLM subagent ACTS (`whats-new`), the driver NAVIGATES routes, so the agent never runs its own BFS. `frontier-cli --emit` surfaces `pendingRoutes` + `routeFrontierStats`, and `decideProgress` gained a `pendingRoutes` input (default 0): `batchLen===0 && pendingRoutes>0` → `visit-route` (drain a queued page), `drained` ONLY when BOTH the template frontier AND the route queue are empty; `/recon` runs `route-cli --visit-next` on a `visit-route` verdict and re-emits, under the same 200-iteration runaway ceiling. `visitRoute` snapshots/discovers but NEVER marks an element explored (Explored ⟺ observed holds — only an act explores). WRITE POSTURE: there is NO read-only mode and no network write-firewall — the crawler is a QA tool on the operator's own stand and is MEANT to create/edit/delete/pay, so an act's writes reach the server. By DEFAULT the deterministic `danger-floor` still declines to CLICK a destructive/auth/payment/communication control and to NAVIGATE to a danger route; EXPLORE-ALL (`BUGHUNTER_EXPLORE_ALL=1`/`--explore-all` + a run id, operator-armed only — `explore-policy.mjs exploreAllArmed`) removes those refusals so every control is classified by actually firing it. ONE rail survives in that mode and is the whole of the safety story: ownership is proven LIVE off the acted handle via the `HUNT-<runId>` marker (`hunt-gate.mjs`), another user's content may be EDITED (the original captured + rolled back through `restore-journal.mjs`) but NEVER destroyed (`FOREIGN_DESTROY`), and an account is deleted only if THIS run created it (`ACCOUNT_PROTECTED`). A fired logout is BRACKETED by an automatic re-login, not refused. THE CRAWLER WAS ITS OWN CEILING, and three defects mattered more than any reach mechanism (all measured on a live React/that router/Radix target, decisions.md 2026-07-19). (1) THE DANGER GATE READS THE AUTHORED ID, not just the accessible name (`danger-floor.authoredIdOf` → the classified testid or the value side of a testid-keyed instanceKey; `normalizeHay` already split kebab/snake/camel). An icon-only logout has an EMPTY name, so the name+route haystack was blank and the gate returned `safe`: the baseline run fired TWO such controls and sent 124 of its remaining 174 navigations to /login, ~70% of the run, while the ledger printed rising coverage. Both are refused now, and the widen catches destructive icon buttons too; guarded BOTH ways, since refusing ordinary authored ids (`employee-save-button`) would delete coverage silently. (2) CLICKABLE ROWS are collected (`dom-snapshot`, a bounded `tr`/`[role=row]`/`li`/`[role=listitem]` + `cursor:pointer` pass named from the LEADING CELL). A row fails the roleless-clickable pass on BOTH its gates — its text is the whole record ("a card") and it contains its own Edit/Delete buttons ("a wrapper") — which is why NO detail page had ever been opened in this project. Rows carry `inRow`, so `listRow` walks ONE representative and counts the rest in drillSkipped. Measured: a row click reaches `/<entity>/<id>`, attributes `GET /api/<entity>/:param`, and reveals 31 controls; one profile page yielded 97. (3) BREADTH INTERLEAVING (`stateful-loop` ROUTE_ACT_BUDGET=20): draining a route to fixpoint before the route queue got a turn meant it never got one (49 routes discovered, 3 visited). The yield falls back to another route with unexplored controls when the queue is empty — the first version only consulted the queue and refunded the budget on the spot, measured at runs of 38 consecutive acts under a declared 20; re-measured after the fix, max run 21. `takeQueuedRoute` is the ONE queue drain, shared by the breadth yield and the terminal path. ALSO: `$id` IS A PARAM SLOT (`graph-store.isParamSegment`/`isParamPath` — that router writes `$id` where React Router writes `:id`, so `/<entity>/$id` was classified static, navigated to LITERALLY, answered 200 by the SPA shell and recorded as a visited section with zero controls); a valid-fill probe respects the DECLARED field type (`input[type=number]` refuses prose outright); `WRITE_VERB` is right-anchored for short verbs ("settings" matched "set" — 3 of 15 reported writes were false, one a plain GET); a redirect to a sign-in page is `SESSION LOST` (loud stop, never a route alias); and a param PATTERN no longer renders as "✗ unreachable" in the ledger. MEASUREMENT DISCIPLINE, learned the hard way: an A/B needs `BUGHUNTER_STATE_DIR` per run and NO overlap — three concurrent runs sharing `state/graph.json` each round adopt and overwrite each other's graph, and every graph-derived comparison from such runs is void (decisions.md RETRACTION, same date). Still unbuilt: depth-N / amortized in-app DFS / depth-cap / per-row instance reveal-paths (a depth-2 chain like open-panel → tab → content; a per-opener replay cap bounding the trusted read-vs-mutation judgment); multi-step / SSO / OAuth / 2FA login + mid-crawl session-expiry re-login (single-form auth IS built, see above), opaque-region persistence in the graph, the Phase-2 designer. Only `recon.md` is native here — the OTHER .claude/agents/*.md are inherited from bughunt-agents and NOT re-wired.

## Commands
```bash
npm test                                                                       # node --test tests/**/*.test.mjs
node lib/recon/whats-new.mjs --url=<url> [--act-template=<id> --fill=<text>]    # keystone: single snapshot + act + report
node lib/recon/recon-run.mjs --url=<url> [--steps=<n>]                          # Phase-1 loop: baseline → drive frontier → report
BUGHUNTER_EXPLORE_ALL=1 BUGHUNTER_RUN_ID=<id> node lib/recon/recon-run.mjs --url=<url> --explore-all  # EXPLORE-ALL: drop the danger-floor refusals and drain the surface FIRING everything (create/edit/delete/pay/call), so every control gets classified. ONE rail: another user's content is edited-with-restore, NEVER destroyed. Needs a run id (the HUNT-<runId> ownership marker)
node lib/recon/certify-loop.mjs --url=<url> [--target=0.95 --max-runs=<n> --steps=<n>]  # GOAL 5: autonomous self-terminating loop — re-crawl (rotating BUGHUNTER_SEED) until Chao2 certifies completeness (Q1→0 / C≥target) or an honest cap
node lib/recon/frontier-cli.mjs --emit [--size=<2-5>]                           # recon agent tool: emit the next receptive-field batch (+ pendingRoutes/routeFrontierStats)

node lib/recon/recon-session.mjs --start|--stop|--status                         # shared browser daemon: ONE chromium for the whole run (whats-new connects to it)
BUGHUNTER_LOGIN_USER=… BUGHUNTER_LOGIN_PASS=… node lib/recon/login.mjs --login-url=<url>  # authed pre-step: login once → state/storage-state.json (0600); then export BUGHUNTER_STORAGE_STATE to crawl logged-in
node lib/recon/report.mjs [--json]                                             # render state/graph.json: honest coverage + causal control→endpoint map
node lib/debug/trace-cli.mjs --open --target=<url> | --close --run=<id>          # bracket a debug-capture run (mint runId / stamp final stats)

# /recon <url>  — slash command (.claude/commands/recon.md): full Phase-1 run — trace open → admin → daemon → recon-agent loop until drained → report
PW_ALLOW_PRIVATE=1 node lib/recon/whats-new.mjs --url=http://127.0.0.1:PORT/    # localhost / fixture targets
BUGHUNTER_STATE_DIR=/tmp/run node lib/recon/whats-new.mjs --url=<url>           # redirect state/ off the repo
```

## Key files
- lib/browser/probe.mjs — causal init-script (fetch/XHR monkeypatch + fire ring, injected pre-navigation)
- lib/browser/causal.mjs — beginCause/endCause walker primitive around the token
- lib/browser/initiator.mjs — CDP initiator classifier; rejects timer/parser-rooted fires
- lib/graph/dom-snapshot.mjs — the single DOM/CSS identity model (template + instance + opaque); `isFrameworkNoiseId` (INC.1) keeps framework-generated wrapper/hashed ids from anchoring a selector (de-fragments antd tabs into instances-of-one-template) OR becoming a sole locator; also classifies a DERIVED per-element `locator` (testid > stable id > role-name > css, with a two-level test-id uniqueness gate) — never an identity input
- lib/graph/graph-store.mjs — state/graph.json nodes (route/element/instance/request) and edges
- lib/browser/session.mjs — launch() (private chromium) + attach() (connect to the shared daemon if up, else cold-launch); one causal wiring. `contextOptions()` is the SINGLE storageState injection point (both newContext sites) — authed crawls load `BUGHUNTER_STORAGE_STATE` here; `opts.anonymous` forces a clean context for login
- lib/recon/step.mjs — the shared browser step: snapshotStep + actStep (the one causal act+capture; attributes revealed controls to the LANDED route, refuses off-origin links)
- lib/recon/scope.mjs — route identity (routeKey: navigable, query/anchor-stripped) + origin scope (sameOrigin, RFC 6454); the multi-route + off-origin-link boundary
- lib/recon/route-frontier.mjs — the BFS URL route queue INSIDE graph.routes (INC.1a): harvestRoutes (read-only a[href] harvest, census-bounded per toUrlPattern) + seedRoutes/visitRoute (snapshot-ONLY page visits — NO addTrigger/causal window, zero phantom edges) + nextPendingRoute (the refill drain) + routeFrontierStats (honest discovered/visited/pending/unreachable/siblingsFolded). The ONE other graph.routes writer, metadata-only
- lib/recon/route-cli.mjs — the /recon route-visit DRIVER (INC.1b): --visit-next drains ONE pending route via visitRoute (attach → snapshot → persist → {visited,routeStats}); the agent-path twin of recon-run's refill hook (the agent ACTS via whats-new, this NAVIGATES routes — snapshot-only, edge-free). Idempotent no-op when nothing is queued
- lib/recon/overlays.mjs — dismissOverlays: curated cookie/consent accept-selector sweep (+ consent-scoped accept-text fallback), run BEFORE the causal window; clears a wall that blocks every underlying control
- lib/recon/whats-new.mjs — single manual step CLI: snapshot → act → report caused + revealed; also harvestRoutes at TWO __idle__ points (post-baseline, post-act endCause) to seed the agent-path route frontier (INC.1b, metadata-only/edge-free)
- lib/recon/frontier.mjs — receptive-field selection (next unexplored templates) + honest denominator
- lib/recon/recon-loop.mjs — Phase-1 loop-driver (pure control-flow; browser step + LLM judge injected)
- lib/recon/recon-run.mjs — loop runner CLI: baseline → persistent step (one browser, re-navigate per act to the target's own route → MULTI-ROUTE) → reconLoop
- lib/recon/browser-daemon.mjs — hosts ONE chromium (launchServer) for a run; publishes state/session.json
- lib/recon/recon-session.mjs — --start/--stop/--status for the daemon + stale-session reaper
- lib/recon/frontier-cli.mjs — the agent's "what next" tool: emit the receptive-field batch + honest stats + the route queue (pendingRoutes/routeFrontierStats, INC.1b) + a progress verdict (continue|visit-route|drained|stalled); no browser
- lib/recon/observe.mjs — the agent's "what I learned" writer: semantics + markExplored; danger-floor backstop
- lib/recon/danger-floor.mjs — deterministic CLASSIFIER + default backstop (destructive/auth/payment/communication); NOT the judge. `REFUSED` set + `routeRefused()` (route-level gate) are the single source shared by step.mjs (click gate) and recon-run.mjs (navigation gate). Under explore-all it still classifies but gates nothing — explore-policy decides
- lib/recon/explore-policy.mjs — the EXPLORE-ALL policy and the ONE rail: `decide()` (pure — name/route/ownership in, a verdict out) permits create/edit/delete/payment/calls and FIRES unknown controls, refusing only `FOREIGN_DESTROY` (another user's content, irreversible) and `ACCOUNT_PROTECTED` (an account this run did not create); flags `needsRestore` (bracket with the restore journal) and `needsRelogin` (the caller re-authenticates after an auth act). `exploreAllArmed()` is the SINGLE arming check — operator env + a run id, never agent-settable
- lib/recon/hunt-gate.mjs — the LIVE ownership proof read off the acted handle: the `HUNT-<runId>` marker (`huntMarker`) via `ownsTarget`/`ownsAnyHunt`/`ownsViaReveal` (portal-dropdown case: ownership is the reveal TRIGGER's item) + `inOwnableItem`. Fail-closed. What makes "ours vs theirs" decidable, so the foreign-content rail depends on it
- lib/recon/restore-journal.mjs — the rollback bracket for an edit to another user's content: capture-before → journal the entry BEFORE the click (a crash still leaves a replayable rollback on disk) → write the original back → close. An unrestored edit stays visible as a pending entry, never silently forgotten
- lib/recon/login.mjs — authed-recon PRE-STEP: env creds (`BUGHUNTER_LOGIN_USER`/`_PASS`) → clean browser → fill form (heuristic + `--*-selector` flags) → VERIFY success → write storageState (0600). Setup, not a measured act: no debug trail, causal attribution untouched. Wired into the crawl via `session.mjs contextOptions()` (`BUGHUNTER_STORAGE_STATE`)
- lib/recon/report.mjs — render the graph: honest coverage + per-route controls + causal control→endpoint map (Phase-2 input)
- lib/debug/trace.mjs — the SINGLE writer of the debug-capture trail: events.ndjson + per-step graph snapshots + before/after key-frames (idle-only, viewport-only, so capture never perturbs causal attribution)
- lib/debug/trace-cli.mjs — bracket a run for /recon: --open (mint runId + run.json) / --close (stamp final coverage stats)
- lib/debug/admin-server.mjs — zero-dep local viewer server: 127.0.0.1-only, loopback Host-guard, no-CORS, resolved-path artifact allowlist
- lib/debug/admin.html — self-contained vanilla page: Walk (timeline + before/after shots + rect box), Graph (route→control→endpoint + step scrubber), Log, Speed; live-polls
- .claude/agents/recon.md — the Sonnet semantic recon subagent (native to this architecture)

- IMPORTANT: reports to the user are SHORT — a few sentences, the conclusion first, one number if it carries the argument. No metric tables, no multi-section write-ups, no restating a finding twice. Detail belongs in decisions.md / docs/CHANGELOG.md, never in chat. Expand only when asked.

- Small files (< 200 lines), single-responsibility.
- Decisions to decisions.md (log rejected alternatives too); changelog to docs/CHANGELOG.md.
